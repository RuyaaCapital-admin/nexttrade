# Contributing to NextTrade

We love your input! We want to make contributing to NextTrade as easy and transparent as possible.

## Development Setup

1. Fork the repo
2. Clone your fork
3. Install dependencies: `npm install`
4. Create a branch: `git checkout -b feature/your-feature`
5. Make your changes
6. Test: `npm run lint`
7. Commit with clear messages
8. Push and open a Pull Request

## Code Standards

- Use meaningful variable names
- Write JSDoc comments for functions
- Follow the existing code style
- Components should be under 200 lines
- Use Tailwind classes for styling

## Component Guidelines

- One component per file
- Export as named export
- Include JSDoc header
- Props validation with TypeScript comments
- Example usage in comments

## Pull Request Process

1. Update README if needed
2. Add tests for new features
3. Ensure all tests pass
4. Update the changelog
5. Request review from maintainers

## Questions?

Open an issue or email support@nexttrade.com
